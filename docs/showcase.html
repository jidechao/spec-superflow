The Problem

AI 编程的两个失控点

用 AI 写代码时，最常碰到的两个问题，恰好分别发生在动手之前和动手之后。

// PAIN POINT 01

方向失控

还没想清楚要做什么，AI 就开始写代码。你说"帮我加个权限控制"，它就开始改几十个文件。改到一半才发现 —— 到底要 RBAC 还是 ABAC？

// PAIN POINT 02

质量失控

规划文档写得明明白白，但执行阶段还是会跑偏。proposal 写了、design 画了，但实现过程中没人盯着测试、没人卡 review，等到合并才发现行为不对。

Design Principles

四大设计原则

Spec First
没有稳定的规划工件，不允许进入实现。把"先想清楚再动手"从口号变成硬约束。
Guarded Handoff
execution-contract.md 是规划到实现的唯一交接层，所有模糊决策在这里被显式压缩为可验证条目。
Strong Guardrails
实现过程中违反契约的行为被明确拦截并回退，而不是靠开发者"感觉不对"来手动纠偏。
Self-Contained
不需要运行时安装 OpenSpec 或 Superpowers，即装即用，零运行时依赖。
Key Innovation

execution-contract.md：不只是一份文档

这是 spec-superflow 区别于其他工作流工具的核心。它不是又一份规划文档，而是一份可检查的契约。

// execution-contract.md

Input / Output 明确本次变更的输入、输出、边界条件
Test Checklist 逐条对应该通过的测试用例
Review Gates 预定义验收关卡和 escalation 规则

让 AI 编程从随机游走变成可控流水线
